A short screencast about Making Software is now up on Amazon. In each of the book's 30 chapters, leading researchers in software engineering explain what we actually know about code review, statistical prediction of software faults, how programmers communicate, whether using design patterns makes code better, the relative quality of open and closed source software, and a host of other topics. We hope you enjoy it.
